Github API: Trees

These methods are relative to the tag 'Trees'

getReference

Get a Tree

Parameter name Description Default Optionnal
user Name of the user undefined false
repo Name of the repository undefined false
sha Tree SHA undefined false
recursive Set to `1` to get the tree recursively undefined true

createTree

Create a Tree. The tree creation API will take nested entries as well. If both a tree and a nested path modifying that tree are specified, it will overwrite the contents of that tree with the new path contents and write a new tree out.

Authentication required

Parameter name Description Default Optionnal
user Name of the user undefined false
repo Name of the repository undefined false
base_tree SHA1 of the tree you want to update with new data undefined true
tree.path File referenced in the tree undefined false
tree.mode File mode - one of `100644` for file (blob), `100755` for executable (blob), `040000` for subdirectory (tree), `160000` for submodule (commit) or `120000` for a blob that specifies the path of a symlink undefined false
tree.type `blob`, `tree`, or `commit` undefined false
tree.sha SHA1 checksum ID of the object in the tree undefined true
tree.content Content you want this file to have - GitHub will write this blob out and use that SHA for this entry. Use either this or `tree.sha` undefined true
