Infoconnect API: People

These methods are relative to the tag 'People'

getPerson

Retrieves details of the individual identified by the resource.

Authentication required

Parameter name Description Default Optionnal
id A twelve digit Infogroup identification number. A number referencing the individual assigned by Infogroup. This number will always refer to a single individual and the reference will never change, unless the resource is permanently removed. Requesting a record by {id} will always return zero or one response. false
resourcetype Identifies the resource type for the response. The default value basic returns a base resource type that is valid across all methods; the resource type parameter is not needed to produce this response. To get a more detailed response set the resource type parameter to enhanced. To retrieve resources with Id's only, set the resource type parameter to core. Additional ResourceTypes are Core, LifeStyle and TargetReady. basic true
Content-Type The content type. application/json false

getPeople

Retrieves lists of people.

Authentication required

Parameter name Description Default Optionnal
resourcetype Identifies the resource type for the response. The default value basic returns a base resource type that is valid across all methods; the resource type parameter is not needed to produce this response. To get a more detailed response set the resource type parameter to enhanced. To retrieve resources with Id's only, set the resource type parameter to core. Additional ResourceTypes are Core, LifeStyle and TargetReady. basic true
city Filters through the People collection by city name. Omaha true
email Filters through the People collection and returns resources with a case-insensitive email address match at the level. johndoe@abcco.com true
emailsha1upper Filters through the People collection and returns resources with a case-insensitive email address match at the level. If used, must be a SHA1 hashed upper case email address. 9d2972490f8c6a9a3a8e052d6050f9d3fde2e54d true
includehistoricalemail IncludeHistoricalEmail can only be used in combination with Email or EmailSha1Upper. IncludeHistoricalEmail is used to retrieve historical records. Including this limits the ResourceType to Basic only. These records do not have an Id therefore cannot be used to return an Enhanced ResourceType. If used, must be True or False. Default is False. true
firstname Filters through the People collection by the primary contact's first name. bill true
lastname Filters through the People collection by the primary contact's last name. doe true
phone Filters through the People collection and returns resources with an ten-digit phone number match. 5558675309 true
postalcode Filters through the People collection by ZIP code or Postal code. 68046 true
stateprovince Filters through the People collection by State or Province abbreviation. NE true
limit Number of responses to return. If used, must be a number between 1 and 10. Default is 10. 10 true
offset Number of responses to be skipped for generating the output. If used, must be a number between 0-100 minus limit. Default is 0. 0 true
Content-Type The content type. application/json false
