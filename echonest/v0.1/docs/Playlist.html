Echonest API: Playlist

These methods are relative to the tag 'Playlist'

playlist.basic

Returns a basic playlist.

[object Object]

Authentication required

Parameter name Description Default Optionnal
callback The callback function for JSONP requests. Required if format is jsonp. true
results The number of results desired. 0 < results < 100 15 true
bucket Specifies which rosetta id space info should be returned. true
limit If 'true', limit the results to any of the given rosetta id space. false true
dmca If true the playlist delivered will meet the DMCA rules. true

playlist.static

Returns a static playlist.

[object Object]

Authentication required

Parameter name Description Default Optionnal
callback The callback function for JSONP requests. Required if format is jsonp. true
type The type of the playlist to be generated. artist true
artist_pick The artist_pick parameter is used to determine how songs are picked for each artist in artist-type playlists. If the asc or desc suffix is ommitted, artist_pick defaults to descending. true
variety The maximum variety of artists to be represented in the playlist. A higher number will allow for more variety in the artists. 0 - 1 (default = 0.3) 0.3 true
distribution Controls the distribution of artists in the playlist. A focused distribution yields a playlist of songs that are tightly clustered around the seeds, whereas a wandering distribution yields a playlist from a broader range of artists. focused true
adventurousness Controls the trade between known music and unknown music. A value of 0 means no adventurousness, only known and preferred music will be played. A value of 1 means high adventurousness, mostly unknown music will be played. A value of auto indicates that the adventurousness should be automatically determined based upon the taste profile of the user. This parameter only applies to catalog and catalog-radio type playlists.0 - 1 (default = 0.2) 0.2 true
results The desired number of songs in the playlist. 0 < results < 100 15 true
max_tempo The maximum tempo for any included songs. 0.0 < tempo < 500.0 (BPM) 500.0 true
min_tempo The minimun tempo for any included songs. 0.0 < tempo < 500.0 (BPM) 0.0 true
max_duration The maximum duration of any song on the playlist. 0.0 < duration < 3600.0 (seconds) 3600.0 true
min_duration The minimun duration of any song on the playlist. 0.0 < duration < 3600.0 (seconds) 0.0 true
max_loudness The maximum loudness of any song on the playlist. -100.0 < loudness < 100.0 (dB) 100.0 true
min_loudness The minimum loudness of any song on the playlist. -100.0 < loudness < 100.0 (dB) -100.0 true
max_danceability The maximum danceability of any song. 0.0 < danceability < 1.0 1.0 true
min_danceability The minimum danceability of any song. 0.0 < danceability < 1.0 0.0 true
max_energy The maximum energy of any song. 0.0 < energy < 1.0 1.0 true
min_energy The minimum energy of any song. 0.0 < energy < 1.0 0.0 true
artist_max_familiarity The maximum artist familiarity for songs in the playlist. 0.0 < familiarity < 1.0 1.0 true
artist_min_familiarity The minimum artist familiarity for songs in the playlist. 0.0 < familiarity < 1.0 0.0 true
artist_max_hotttnesss The maximum artist hotttness for songs in the playlist. 0.0 < hotttnesss < 1.0 1.0 true
artist_min_hotttnesss The minimum artist hotttness for songs in the playlist. 0.0 < hotttnesss < 1.0 0.0 true
artist_start_year_before Matches artists that have an earliest start year before the given value. Examples: 1970, 2011, present true
artist_start_year_after Matches artists that have an earliest start year after the given value. Examples: 1970, 2011, present true
artist_end_year_before Matches artists that have a latest end year before the given value. Examples: 1970, 2011, present true
artist_end_year_after Matches artists that have a latest end year after the given value. Examples: 1970, 2011, present true
song_max_hotttnesss The maximum hotttnesss for songs in the playlist. 0.0 < hotttnesss < 1.0 1.0 true
song_min_hotttnesss The minimum hotttnesss for songs in the playlist. 0.0 < hotttnesss < 1.0 0.0 true
max_longitude The maximum longitude for the location of artists in the playlist. -180.0 < longitude < 180.0 180.0 true
min_longitude The minimum longitude for the location of artists in the playlist. -180.0 < longitude < 180.0 -180.0 true
max_latitude The maximum latitude for the location of artists in the playlist. -90.0 < latitude < 90.0 90.0 true
min_latitude The minimum latitude for the location of artists in the playlist. -90.0 < latitude < 90.0 -90.0 true
mode The mode of songs in the playlist. (minor, major) 0, 1 true
key The key of songs in the playlist. (c, c-sharp, d, e-flat, e, f, f-sharp, g, a-flat, a, b-flat, b) 0 - 11 true
bucket Indicates what data should be returned in the playlist (for json and xml types only). true
sort Indicates how the songs should be ordered in the playlist. true
limit If 'true', limit the results to any of the given idspaces or catalogs. false true
dmca If true the playlist delivered will meet the DMCA rules. true

dmca If true the playlist delivered will meet the DMCA rules. true

chain_xspf If true, an additional 'track' is added to the end XSPF playlist that is a URL to the next XSPF in the chain. true
steer Using the previously played track as the basis, make all future tracks some multiplier of the selected attribute(s). required terms use a multiplier in a boost-like format - e.g. energy^.5 to make all future songs have half the energy of the previously played song, or loudness^1.3 to add 30%. true
steer_description Using the current playlist as a basis, add or remove terms relating to the artists and songs to be played. Applying a term again will overwrite the previous term, including boost. Boosting with a "0" will remove a term from the list of qualifiers. Using 'description' rather than 'steer_description' will clear the description list and restart the playlist session with the new term. true
steer_style Using the current playlist as a basis, add or remove terms relating to the style of artists and songs to be played. Applying a style again will overwrite the previous style, including boost. Boosting with a "0" will remove a term from the list of qualifiers. Using 'style' rather than 'steer_style' will clear the style list and restart the playlist session with the new style. true
steer_mood Using the current playlist as a basis, add or remove terms relating to the mood of artists and songs to be played. Applying a mood again will overwrite the previous mood, including boost. Boosting with a "0" will remove a mood from the list of moods. Using 'mood' rather than 'steer_mood' will clear the mood list and restart the playlist session with the new mood. true
ban Banning will prevent the song/all songs by that artist from appearing again in the current playlist session. true
