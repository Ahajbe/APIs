Echonest API: Song

These methods are relative to the tag 'Song'

song.search

Search for songs given different query types.

[object Object]

Authentication required

Parameter name Description Default Optionnal
title The title of the song. true
artist The name of the primary artist. true
combined Query both artist and title fields. true
description A description of the artist. true
style A musical style or genre like rock, jazz, or funky. true
mood A mood like happy or sad. true
rank_type For search by description, style or mood indicates whether results should be ranked by query relevance or by artist familiarity relevance true
artist_id The artist ID. An Echo Nest ID or a Rosetta ID (See http://developer.echonest.com/docs/v4/index.html#rosetta). true
results The number of results desired. 0 < results < 100 15 true
start The desired index of the first result returned. 15 true
max_tempo The maximum tempo for the song. 0.0 < tempo < 500.0 (BPM) 500.0 true
min_tempo The minimun tempo for the song. 0.0 < tempo < 500.0 (BPM) 0.0 true
max_duration The maximum duration of any song. 0.0 < duration < 3600.0 (seconds) 3600.0 true
min_duration The minimun duration of any song. 0.0 < duration < 3600.0 (seconds) 0.0 true
max_loudness The maximum loudness of any song. -100.0 < loudness < 100.0 (dB) 100.0 true
min_loudness The minimum loudness of any song. -100.0 < loudness < 100.0 (dB) -100.0 true
artist_max_familiarity The maximum familiarity of any song. 0.0 < familiarity < 1.0 1.0 true
artist_min_familiarity The minimum familiarity of any song. 0.0 < familiarity < 1.0 0.0 true
artist_start_year_before Matches artists that have an earliest start year before the given value. Examples: 1970, 2011, present true
artist_start_year_after Matches artists that have an earliest start year after the given value. Examples: 1970, 2011, present true
artist_end_year_before Matches artists that have a latest end year before the given value. Examples: 1970, 2011, present true
artist_end_year_after Matches artists that have a latest end year after the given value. Examples: 1970, 2011, present true
song_max_hotttnesss The maximum hotttnesss of any song. 0.0 < hotttnesss < 1.0 1.0 true
song_min_hotttnesss The minimum hotttnesss of any song. 0.0 < hotttnesss < 1.0 0.0 true
artist_max_hotttnesss The maximum hotttnesss of any song's artist. 0.0 < hotttnesss < 1.0 1.0 true
artist_min_hotttnesss The minimum hotttnesss of any song's artist. 0.0 < hotttnesss < 1.0 0.0 true
max_longitude The maximum longitude of the primary artist location. -180.0 < longitude < 180.0 180.0 true
min_longitude The minimum longitude of the primary artist location. -180.0 < longitude < 180.0 -180.0 true
max_latitude The maximum latitude of the primary artist location. -90.0 < latitude < 90.0 90.0 true
min_latitude The minimum latitude of the primary artist location. -90.0 < latitude < 90.0 -90.0 true
max_danceability The maximum danceability of any song. 0.0 < danceability < 1.0 1.0 true
min_danceability The minimum danceability of any song. 0.0 < danceability < 1.0 0.0 true
max_energy The maximum energy of any song. 0.0 < energy < 1.0 1.0 true
min_energy The minimum energy of any song. 0.0 < energy < 1.0 0.0 true
mode The mode of songs. (minor, major) 0, 1 true
key The key of songs in the playlist. (c, c-sharp, d, e-flat, e, f, f-sharp, g, a-flat, a, b-flat, b) 0 - 11 true
bucket Indicates what data should be returned for each song. If specifying the "tracks" bucket, a bucket with an id space must also be specified. See http://developer.echonest.com/docs/v4/index.html#rosetta for details on retrieving information associated with a particular catalog. Bucket:
audio_summary: returns summary audio parameters for the song
artist_familiarity: returns the familiarity for the song's artist
artist_hotttnesss: returns the hotttnesss for the song's artist
artist_location: returns information about the location of origin for the song's artist
song_hotttnesss: returns the hotttnesss of the song
tracks: returns detailed track information for the song. You must also specify a Rosetta id space such as 7digital-US.
id:rosetta-catalog: returns catalog specific information about the song for the given catalog. See http://developer.echonest.com/docs/v4/index.html#rosetta for details.
id:Personal-Catalog-ID: returns personal catalog specific information about the song for the given catalog. See http://developer.echonest.com/docs/v4/index.html#rosetta for details.		 true
sort Indicates how the songs results should be ordered. true
limit If 'true', limit the results to any of the given idspaces or catalogs. false true

song.identify.get

Identifies a song given an Echoprint or Echo Nest Musical Fingerprint hash codes.

[object Object]

Authentication required

Parameter name Description Default Optionnal
version Version of codegen used to generate the code. Currently supported versions are 3.15 for ENMFP and 4.11 for Echoprint. If not specified, version 3.15 is used. See the section below on Echo Nest Fingerprinting systems to determine which fingerprinting system is best for you. true
bucket Specifies which information should be returned for each matching song. Bucket:
audio_summary: returns summary audio parameters for the song
artist_familiarity: returns the familiarity for the song's artist
artist_hotttnesss: returns the hotttnesss for the song's artist
artist_location: returns information about the location of origin for the song's artist
song_hotttnesss: returns the hotttnesss of the song
tracks: returns detailed track information for the song. You must also specify a Rosetta id space such as 7digital-US.
id:rosetta-catalog: returns catalog specific information about the song for the given catalog. See http://developer.echonest.com/docs/v4/index.html#rosetta for details.
id:Personal-Catalog-ID: returns personal catalog specific information about the song for the given catalog. See http://developer.echonest.com/docs/v4/index.html#rosetta for details.		 true
code The FP hashcodes for the track. true
artist The name of the artist from the ID3 tag. true
title The title of the track from the ID3 tag. true
release The title of the album/release from the ID3 tag. true
duration The length of time of the track, in seconds. true
genre The genre from the ID3 tag. true
