Twitter API: Favorite

These methods are relative to the tag 'Favorite'

favorites

Returns the 20 most recent Tweets favorited by the authenticating or specified user. If you do not provide either a user_id or screen_name to this method, it will assume you are requesting on behalf of the authenticating user. Specify one or the other for best results.

Parameter name Description Default Optionnal
user_id The ID of the user for whom to return results for. true
screen_name The screen name of the user for whom to return results for. true
count Specifies the number of records to retrieve. Must be less than or equal to 200. Defaults to 20. true
since_id Returns results with an ID greater than (that is, more recent than) the specified ID. There are limits to the number of Tweets which can be accessed through the API. If the limit of Tweets has occured since the since_id, the since_id will be forced to the oldest ID available. true
max_id Returns results with an ID less than (that is, older than) or equal to the specified ID. true
include_entities The entities node will be omitted when set to false. true
page true

favoriteStatus

Favorites the status specified in the ID parameter as the authenticating user. Returns the favorite status when successful.

This process invoked by this method is asynchronous. The immediately returned status may not indicate the resultant favorited status of the tweet. A 200 OK response from this method will indicate whether the intended action was successful or not.

Parameter name Description Default Optionnal
id The numerical ID of the desired status. false
include_entities The entities node will be omitted when set to false. true

unfavoriteStatus

Un-favorites the status specified in the ID parameter as the authenticating user. Returns the un-favorited status in the requested format when successful.

This process invoked by this method is asynchronous. The immediately returned status may not indicate the resultant favorited status of the tweet. A 200 OK response from this method will indicate whether the intended action was successful or not.

Parameter name Description Default Optionnal
id The numerical ID of the desired status. false
include_entities The entities node will be omitted when set to false. true
