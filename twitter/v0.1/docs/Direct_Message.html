Twitter API: Direct Message

These methods are relative to the tag 'Direct Message'

directMessages

Returns the 20 most recent direct messages sent to the authenticating user. Includes detailed information about the sender and recipient user. You can request up to 200 direct messages per call, up to a maximum of 800 incoming DMs.

Parameter name Description Default Optionnal
format json false
count Specifies the number of direct messages to try and retrieve, up to a maximum of 200. The value of count is best thought of as a limit to the number of Tweets to return because suspended or deleted content is removed after the count has been applied. undefined true
max_id Returns results with an ID less than (that is, older than) or equal to the specified ID. undefined true
page Specifies the page of results to retrieve. undefined true
since_id Returns results with an ID greater than (that is, more recent than) the specified ID. There are limits to the number of Tweets which can be accessed through the API. If the limit of Tweets has occured since the since_id, the since_id will be forced to the oldest ID available. undefined true
include_entities The entities node will not be included when set to false. undefined true
skip_status When set to either true, t or 1 statuses will not be included in the returned user objects. undefined true

directMessagesSent

Returns the 20 most recent direct messages sent by the authenticating user. Includes detailed information about the sender and recipient user. You can request up to 200 direct messages per call, up to a maximum of 800 outgoing DMs. Important: This method requires an access token with RWD (read, write & direct message) permissions. Consult The Application Permission Model for more information.

format json false
count Specifies the number of direct messages to try and retrieve, up to a maximum of 200. The value of count is best thought of as a limit to the number of Tweets to return because suspended or deleted content is removed after the count has been applied. undefined true
max_id Returns results with an ID less than (that is, older than) or equal to the specified ID. undefined true
page Specifies the page of results to retrieve. undefined true

If the limit of Tweets has occured since the since_id, the since_id will be forced to the oldest ID available. undefined true

showDirectMessage

Returns a single direct message, specified by an id parameter. Like the /1.1/direct_messages.format request, this method will include the user objects of the sender and recipient. Important: This method requires an access token with RWD (read, write & direct message) permissions. Consult The Application Permission Model for more information.

Parameter name Description Default Optionnal
format json false
id The ID of the direct message. undefined true

newDirectMessage

Sends a new direct message to the specified user from the authenticating user. Requires both the user and text parameters and must be a POST. Returns the sent message in the requested format if successful.

Parameter name Description Default Optionnal
format json false
user_id The ID of the user who should receive the direct message. Helpful for disambiguating when a valid user ID is also a valid screen name. undefined true
screen_name The screen name of the user who should receive the direct message. Helpful for disambiguating when a valid screen name is also a user ID. undefined true
text The text of your direct message. Be sure to URL encode as necessary, and keep the message under 140 characters. undefined true

destroyDirectMessage

Destroys the direct message specified in the required ID parameter. The authenticating user must be the recipient of the specified direct message. Important: This method requires an access token with RWD (read, write and direct message) permissions. Consult The Application Permission Model for more information.

Parameter name Description Default Optionnal
format json false
id The ID of the direct message to delete. undefined false
include_entities The entities node will not be included when set to false. undefined true
