Stackexchange API: Comments

These methods are relative to the tag 'Comments'

commentsList

Gets all the comments on the site. If you're filtering for interesting comments (by score, creation date, etc.) make use of the sort paramter with appropriate min and max values. If you're looking to query conversations between users, instead use the /users/{id}/mentioned and /users/{id}/comments/{toid} methods.

Parameter name Description Default Optionnal
fromdate Unix timestamp of the minimum creation date on a returned item. undefined true
max Maximum of the range to include in the response according to the current sort. undefined true
min Minimum of the range to include in the response according to the current sort. undefined true
order How the current sort should be ordered. One of desc (default), or asc. undefined true
page The pagination offset for the current collection. Affected by the specified pagesize. undefined true
pagesize The number of collection results to display during pagination. Should be between 0 and 100 inclusive. undefined true
sort How a collection should be sorted. One of desc (default), or asc. undefined true
todate Unix timestamp of the maximum creation date on a returned item. undefined true

commentsGet

Gets the comments on the posts identified in id, regardless of the type of the posts. This method is meant for cases when you are unsure of the type of the post id provided. Generally, this would be due to obtaining the post id directly from a user.

deleteComment

Deletes a comment. Use an access_token with write_access to delete a comment. In practice, this method will never return an object.

Parameter name Description Default Optionnal
id The comment id. undefined false
preview [object Object] undefined true

editComment

Edit an existing comment. Use an access_token with write_access to edit an existing comment. This method return the created comment.

Parameter name Description Default Optionnal
id The comment id. undefined false
