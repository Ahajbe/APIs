Stackexchange API: Search

These methods are relative to the tag 'Search'

search

Searches a site for any questions which fit the given criteria.

This method is intentionally quite limited. For more general searching, you should use a proper internet search engine restricted to the domain of the site in question.

At least one of tagged or intitle must be set on this method. nottagged is only used if tagged is also set, for performance reasons.

Parameter name Description Default Optionnal
nottagged List of tags delimited by semi-colons that must not be on a question.
This is a vectorized parameter, multiple ids can be sent a single request if delimited with a semi-colon.		 true
tagged List of tags delimited by semi-colons of which at least one must be on a question.
This is a vectorized parameter, multiple ids can be sent a single request if delimited with a semi-colon.		 true
intitle A string that must appear verbatim in the title of a question. true
max Maximum of the range to include in the response according to the current sort. true
min Minimum of the range to include in the response according to the current sort. true
order How the current sort should be ordered. One of desc (default), or asc. true
page The pagination offset for the current collection. Affected by the specified pagesize. true
pagesize The number of collection results to display during pagination. Should be between 0 and 100 inclusive. true
sort he sorts accepted by this method operate on the follow fields of the question object:

activity – last_activity_date
creation – creation_date
votes – score
relevance – matches the relevance tab on the site itself
Does not accept min or max
activity is the default sort.
 true
site To specified on which site you want to search. stackoverflow (default) stackoverflow false

advancedSearch

Searches a site for any questions which fit the given criteria.

Search criteria are expressed using the following parameters:

Parameter name Description Default Optionnal
nottagged List of tags delimited by semi-colons that must not be on a question.
This is a vectorized parameter, multiple ids can be sent a single request if delimited with a semi-colon.		 true
tagged List of tags delimited by semi-colons of which at least one must be on a question.
This is a vectorized parameter, multiple ids can be sent a single request if delimited with a semi-colon.		 true
title text which must appear in returned questions' titles. true
max Maximum of the range to include in the response according to the current sort. true
min Minimum of the range to include in the response according to the current sort. true
order How the current sort should be ordered. One of desc (default), or asc. true
page The pagination offset for the current collection. Affected by the specified pagesize. true
pagesize The number of collection results to display during pagination. Should be between 0 and 100 inclusive. true
sort he sorts accepted by this method operate on the follow fields of the question object:

activity – last_activity_date
creation – creation_date
votes – score
relevance – matches the relevance tab on the site itself
Does not accept min or max
activity is the default sort.
 true
site To specified on which site you want to search. stackoverflow (default) stackoverflow false
q a free form text parameter, will match all question properties based on an undocumented algorithm. true
accepted true to return only questions with accepted answers, false to return only those without. Omit to elide constraint. true
answers the minimum number of answers returned questions must have. true
body text which must appear in returned questions' bodies. true
closed true to return only closed questions, false to return only open ones. Omit to elide constraint. true
migrated true to return only questions migrated away from a site, false to return only those not. Omit to elide constraint. true
notice true to return only questions with post notices, false to return only those without. Omit to elide constraint. true
user the id of the user who must own the questions returned. true
url a url which must be contained in a post, may include a wildcard. true
views the minimum number of views returned questions must have. true
wiki true to return only community wiki questions, false to return only non-community wiki ones. Omit to elide constraint true

similar

Returns questions which are similar to a hypothetical one based solely on a title.

This method is roughly equivalent to a site's related questions suggestion on the ask page.

This method is useful for correlating data outside of a Stack Exchange site with similar content within one.
