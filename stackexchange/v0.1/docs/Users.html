Stackexchange API: Users

These methods are relative to the tag 'Users'

usersList

Returns all users on a site.

Sorting and min/max ranges allow for interesting queries against users. For instance, you can find how many users have between 1,000 and 2,000 reputation with the parameters sort=reputation min=1000 max=2000 pagesize=0.

Parameter name Description Default Optionnal
filter Required text in returned tags. true
fromdate Unix timestamp of the minimum creation date on a returned item. true
max Maximum of the range to include in the response according to the current sort. true
min Minimum of the range to include in the response according to the current sort. true
order How the current sort should be ordered. One of desc (default), or asc. true
page The pagination offset for the current collection. Affected by the specified pagesize. true
pagesize The number of collection results to display during pagination. Should be between 0 and 100 inclusive. true
sort The sorts accepted by this method operate on the follow fields of the user object:

reputation – reputation
creation – creation_date
name – display_name
modified – last_modified_date
reputation is the default sort.
 true
todate Unix timestamp of the maximum creation date on a returned item. true
inname The inname parameter lets consumers filter the results down to just those users with a certain substring in their display name. For example, inname=kevin will return all users with both users named simply "Kevin" or those with Kevin as one of (or part of) their names; such as "Kevin Montrose". true
site To specified on which site you want to search. stackoverflow (default) stackoverflow false

userMergesGet

Returns a record of merges that have occurred involving the passed account ids.

This method allows you to take now invalid account ids and find what account they've become, or take currently valid account ids and find which ids were equivalent in the past.

This is most useful when confirming that an account_id is in fact "new" to an application.

Account merges can happen for a wide range of reasons, applications should not make assumptions that merges have particular causes.

Note that accounts are managed at a network level, users on a site may be merged due to an account level merge but there is no guarantee that a merge has an effect on any particular site.

This method returns a list of account_merge.

id A single primary key identifier or a vectorized, semicolon-delimited list of identifiers.
