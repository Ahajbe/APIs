Pivotaltracker API: Manage Stories

These methods are relative to the tag 'Manage Stories'

postAddStory

Adds a new story

Authentication required

Parameter name Description Default Optionnal
PROJECT_ID The ID of the project
 false

putStory

Updates a story

Authentication required

Parameter name Description Default Optionnal
PROJECT_ID The ID of the project
 false
STORY_ID The ID of the story
 false

deleteStory

Deletes a story

Authentication required

Parameter name Description Default Optionnal
PROJECT_ID The ID of the project
 false
STORY_ID The ID of the story
 false

Authentication required

Parameter name Description Default Optionnal
PROJECT_ID The ID of the project
 false
STORY_ID The ID of the story
 false

putDeliverAllFinished

Takes all finished stories and marks them as delivered. This could be used to automate a demo deploy process. The updated stories are returned as the result.

Authentication required

Parameter name Description Default Optionnal
PROJECT_ID The ID of the project
 false

postMoveStories

Moves a story before or after another story. The moved story is returned in the response.

Authentication required

Parameter name Description Default Optionnal
PROJECT_ID The ID of the project
 false
STORY_ID The ID of the story
 false
