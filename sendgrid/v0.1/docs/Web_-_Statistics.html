Sendgrid API: Web - Statistics

These methods are relative to the tag 'Web - Statistics'

stats.get

Retrieves statistics. If provided with no parameters the current day���s statistics are returned instead. Note that you can use either the days parameter or the start_date and end_date parameter.

Authentication required

Parameter name Description Default Optionnal
days Number of days in the past to include statistics (includes today). Must be an integer greater than 0. true
start_date The start date to look up statistics. Date must be in YYYY-MM-DD format and be before the end_date parameter. true
end_date The end date to look up statistics. Date must be in YYYY-MM-DD format and be after the start_date parameter. true

stats.aggregates.get

Retrieves all-time totals for your subusers.

Authentication required

Parameter name Description Default Optionnal
aggregate This is used to indicate you are interested in all time totals. Must be set to 1. false

stats.categories.get

Retrieves a list of all the categories used in your account.

Authentication required

Parameter name Description Default Optionnal
list Whether or not to list categories. Must be set to true. false

stats.category.stats.get

Retrieves statistics broken down by category. If the category does not exist, there will be an empty result set. Note that you can use either the days parameter or the start_date and end_date parameter.

Authentication required

Parameter name Description Default Optionnal
category The category you will specify to retrieve detailed stats. Must be an existing category that has statistics. This can also be passed in as an array, to send to multiple categories (i.e. category[]={category1}&category[]={category2}&...&category[]={categoryN)). false
days Number of days in the past to include statistics (includes today). Must be an integer greater than 0. true
start_date The start date to look up statistics. Date must be in YYYY-MM-DD format and be before the end_date parameter. true
end_date The end date to look up statistics. Date must be in YYYY-MM-DD format and be after the start_date parameter. true
