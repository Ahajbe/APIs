Sendgrid API: Web - Spam Reports

These methods are relative to the tag 'Web - Spam Reports'

spamreports.get

Retrieves a list of spam reports with addresses and response codes, optionally with dates.

Authentication required

Parameter name Description Default Optionnal
date Retrieve the timestamp of the spam report records. It will return a date in a MySQL timestamp format ��� YYYY-MM-DD HH:MM:SS. Must be set to 1. true
days Number of days in the past for which to retrieve spam reports (includes today). If specified, must be an integer greater than 0. true
start_date The start of the date range for which to retrieve spam reports. Date must be in YYYY-MM-DD format and be earlier than the end_date parameter. true
end_date The end of the date range for which to retrieve spam reports. Date must be in YYYY-MM-DD format and be later than the start_date parameter. true
limit Limit the number of results returned. true
offset Beginning point in the list to retrieve from. true
email Email addresses to search for. true

spamreports.delete

Deletes an address from the Spam Reports list.

Authentication required

Parameter name Description Default Optionnal
email Email spam reports address to remove. Must be a valid user account email. false
start_date The date to start deleting at. Date must be in YYYY-mm-dd format and be before the end_date parameter. true
end_date The date to end deleting at. Date must be in YYYY-mm-dd format and be after the start_date parameter. true

spamreports.count

Counts the total number of spam reports.

Authentication required

Parameter name Description Default Optionnal
start_date The date to start counting from. Date must be in YYYY-mm-dd format and be before the end_date parameter. true
end_date The date to end counting from. Date must be in YYYY-mm-dd format and be after the start_date parameter. true
