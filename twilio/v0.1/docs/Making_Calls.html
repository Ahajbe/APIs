Twilio API: Making Calls

These methods are relative to the tag 'Making Calls'

makeCall

Make a call.

Parameter name Description Default Optionnal
format json false
AccountSid Represents the unique id of the Account responsible for this caller Id undefined false
From The phone number to use as the caller id. Format with a '+' and country code e.g., +16175551212 (E.164 format). Must be a Twilio number or a valid outgoing caller id for your account. undefined false
To The number to call formatted with a '+' and country code e.g., +16175551212 (E.164 format). Twilio will also accept unformatted US numbers e.g., (415) 555-1212, 415-555-1212. undefined false
Url The fully qualified URL that should be consulted when the call connects. Just like when you set a URL for your inbound calls. undefined false
Method The HTTP method Twilio should use when requesting the required Url parameter's value above. Defaults to POST. undefined true
FallbackUrl A URL that Twilio will request if an error occurs requesting or executing the TwiML at Url. undefined true
FallbackMethod The HTTP method that Twilio should use to request the FallbackUrl. Must be either GET or POST. Defaults to POST. undefined true
StatusCallback A URL that Twilio will request when the call ends to notify your app. undefined true
StatusCallbackMethod The HTTP method Twilio should use when requesting the above URL. Defaults to POST. undefined true
SendDigits A string of keys to dial after connecting to the number. Valid digits in the string include: any digit (0-9), '#' and '*'. For example, if you connected to a company phone number, and wanted to dial extension 1234 and then the pound key, use SendDigits=1234#. Remember to URL-encode this string, since the '#' character has special meaning in a URL. undefined true
IfMachine Tell Twilio to try and determine if a machine (like voicemail) or a human has answered the call. Possible values are Continue and Hangup. See the answering machines section below for more info. undefined true
Timeout The integer number of seconds that Twilio should allow the phone to ring before assuming there is no answer. Default is 60 seconds, the maximum is 999 seconds. Note, you could set this to a low value, such as 15, to hangup before reaching an answering machine or voicemail. Also see the answering machine section for other solutions. undefined true
