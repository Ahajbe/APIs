Default
Description
Parameter
10000
string

The maximum number of ad units to include in the response, used for paging. The minimum value is 0 (zero) and the maximum value is 10000 (10 thousand). Default is 10000 (10 thousand).

maxResults

Optional

string

A continuation token, used to page through ad units. To retrieve the next page, set this parameter to the value of "nextPageToken" from the previous response. Default returns the first page of results.

pageToken

Optional

string

Name of the JavaScript callback function that handles the response. Used in JavaScript JSON-P requests.

Optional

Optional

false
string

Returns the response in a human-readable format (with indentations and line breaks) if true. When this is false, it can reduce the response payload size, which might lead to better performance in some environments. Default value: false.

prettyPrint

Optional

string

Lets you enforce per-user quotas from a server-side application even in cases when the user's IP address is unknown. This can occur, for example, with applications that run cron jobs on App Engine on a user's behalf. You can choose any arbitrary string that uniquely identifies a user, but it is limited to 40 characters. Alternative to userIp, overrides userIp if both are provided.

quotaUser

Optional

string

IP address of the end user for whom the API call is being made. Lets you enforce per-user quotas when calling the API from a server-side application.
